Since the goal of the presentation layer is that can be detached from the core logic layer, it has its own config.php file. This file tells the presentation layer where to find the Core layer and how to communicate with it. There are two modes of communication. The first mode is collapsed and means that the Core is in the same path as the presentation layer. This is the most common and currently only supported method.
Note:
If you turn collapsed mode off, the system enters the other mode where the API and Core can be in a different location or even on a different server. Currently this mode is not supported.
